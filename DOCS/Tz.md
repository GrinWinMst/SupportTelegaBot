# Техническое задание (ТЗ) на разработку Telegram-бота "SupportDawn"

## 1. Общие сведения
**Название проекта:** SupportDawn
**Назначение:** Автоматизация технической поддержки игроков и администрирование игрового сервера через Telegram.

## 2. Цели проекта
1.  Обеспечение оперативной связи между игроками и администрацией.
2.  Упрощение процесса выдачи внутриигровых наград (Daily Rewards, Kits).
3.  Предоставление удобного интерфейса для модерации (баны, логи) без необходимости захода на сервер.
4.  Автоматизация уведомлений о событиях на сервере.

## 3. Роли пользователей
- **Игрок:** Может создавать тикеты в поддержку, получать награды, просматривать правила.
- **Саппорт:** Обрабатывает тикеты, общается с игроками через интерфейс бота.
- **Администратор:** Имеет полный доступ к логам, управлению банами, настройкам системы и RCON-командам.

## 4. Функциональные требования

### 4.1. Система тикетов (Поддержка)
- Создание обращения игроком.
- Уведомление персонала о новом тикете.
- Чат-система между игроком и саппортом внутри бота.
- Закрытие тикета с сохранением истории.

### 4.2. Администрирование и модерация
- Система блокировок (бан/разбан) по ID или никнейму.
- Просмотр логов действий персонала и системных событий.
- Расширенная статистика по обращениям.

### 4.3. Игровая интеграция (RCON)
- Выполнение команд на сервере через RCON-протокол.
- Выдача игровых наборов (PVP-киты).
- Автоматическая выдача ежедневных наград.

### 4.4. База данных
- Хранение данных о пользователях, тикетах, наградах и логах.
- Поддержка SQLite (для локальной разработки) и MySQL (для продакшена).

## 5. Технические требования
- **Язык разработки:** Python 3.10+
- **Фреймворк:** aiogram 3.x (асинхронная работа).
- **СУБД:** MySQL 8.0 / SQLite 3.
- **Среда запуска:** Linux (Ubuntu/Debian) или Windows (для тестов).

## 6. Требования к интерфейсу
- Использование Inline и Reply клавиатур для навигации.
- Интуитивно понятное меню для игроков.
- Раздельные панели управления для саппортов и админов.

## 7. Безопасность
- Хранение токенов и паролей в зашифрованном виде или переменных окружения (`.env`).
- Валидация прав доступа для каждой команды.
- Защита от спама (rate limiting).
